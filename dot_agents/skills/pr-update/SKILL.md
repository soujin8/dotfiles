# Pull Request Update

`gh` コマンドで現在のブランチの Pull Request の内容を更新する。

## 手順

1. `git status --short` と現在のブランチを確認する。
2. `gh pr view` で現在のブランチに紐づく Pull Request の詳細を確認する。
3. PR テンプレートがある場合は、テンプレートの構成に従って本文を更新する。
4. PR 本文には「変更の意図」のセクションを作り、なぜその変更が必要なのかを明確に書く。
5. 内容に不明点がある場合はユーザーに質問し、回答を本文に反映する。
6. 更新内容を確認してから、`gh pr edit --body-file` などで PR 本文を更新する。

## 注意

- PR の既存本文に重要な情報がある場合は消さずに統合する。
- 差分や既存説明から分からない意図は推測で埋めない。
- PR テンプレートのチェックリストや必須項目は維持する。
