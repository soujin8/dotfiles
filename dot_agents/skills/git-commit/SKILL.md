# Git Commit

現在 git staging にある内容を元にして、コミットメッセージを作成する。

## 手順

1. `git status --short` で staging 済みのファイルを確認する。
2. `git diff --cached` で staged diff を読み、変更の意図と影響範囲を把握する。
3. staged されていない変更は、ユーザーから明示されない限りコミット対象に含めない。
4. 差分の内容に基づいて、簡潔で具体的なコミットメッセージを作成する。
5. ユーザーがコミット実行まで求めている場合は、そのメッセージで `git commit` を実行する。

## メッセージ方針

- 1 行目は変更内容を命令形または簡潔な説明で書く。
- 必要に応じて本文に背景、変更点、注意点を箇条書きで追加する。
- 差分から意図が読み取れない場合は、推測で確定せずユーザーに確認する。
